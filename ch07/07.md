### 函数装饰器和闭包

- 函数装饰器，在源码中标记函数，用某种方式增强函数的行为

- 装饰器通常把函数替换成另一个函数

  ```python
  def deco(func):
      def inner():
          print('inner running')
  
      return inner
  
  
  # 使用deco 装饰target
  @deco
  def target():
      print('running target')
  
  
  print(target())
  # inner running
  
  # target 现在是inner的引用
  print(target)
  # <function deco.<locals>.inner at 0x000001F4D75F2488>
  ```

  
- 函数装饰器在加载模块的时候立即执行，而被装饰的函数在只在明确调用时运行

  ```python
  registry = []
  
  
  def register(func):
      print('running register{0}'.format(func))
      registry.append(func)
  
      return func
  
  
  @register
  def f1():
      print('running f1')
  
  
  @register
  def f2():
      print('running f2')
  
  
  def f3():
      print('running f3')
  
  
  print('running start')
  print('registry ', registry)
  f1()
  f2()
  f3()
  
  # out
  running register<function f1 at 0x000001BDA3DA2510>
  running register<function f2 at 0x000001BDA3DA2598>
  running start
  registry  [<function f1 at 0x000001BDA3DA2510>, <function f2 at 0x000001BDA3DA2598>]
  running f1
  running f2
  running f3
  ```



- dis模块反汇编

  def f5():
      print('running f5')
  
  
  print(dis(f5))

   68           0 LOAD_GLOBAL              0 (print)
                2 LOAD_CONST               1 ('running f5')
